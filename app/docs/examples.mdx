# Examples

These examples show what the obfuscation flow is doing and why `token.css` matters when you use design-token utilities.

## Standard utility string

### Source component

```tsx
<div className="flex items-center justify-center p-4 bg-blue-500 text-white rounded-lg shadow-lg">
  <h1 className="text-2xl font-bold">Hello World</h1>
</div>
```

### Obfuscated result

```tsx
<div className="a1b2c3d4">
  <h1 className="e5f6g7h8">Hello World</h1>
</div>
```

### Generated CSS

```css
.a1b2c3d4 {
  @apply flex items-center justify-center p-4 bg-blue-500 text-white rounded-lg shadow-lg;
}

.e5f6g7h8 {
  @apply text-2xl font-bold;
}
```

## Token-based utility string

This is the important case for projects that use semantic theme classes.

### Source component

```tsx
<button className="bg-primary text-primary-foreground border border-border hover:bg-primary/90">
  Save changes
</button>
```

### Required token file

```css
/* app/token.css */
@theme inline {
  --color-primary: var(--primary);
  --color-primary-foreground: var(--primary-foreground);
  --color-border: var(--border);
}

:root {
  --primary: oklch(0.205 0 0);
  --primary-foreground: oklch(0.985 0 0);
  --border: oklch(0.922 0 0);
}
```

### Required generator references

```js
css += '@reference "tailwindcss";\n';
css += '@reference "./token.css";\n\n';
```

### Generated CSS

```css
.z9y8x7w6 {
  @apply bg-primary text-primary-foreground border border-border hover:bg-primary/90;
}
```

If you skip `token.css` or forget the `@reference "./token.css";` line, utilities like `bg-primary` and `bg-background` are much more likely to fail in the generated stylesheet.

## Example project structure

```text
app/
├── docs/
│   ├── examples.mdx
│   └── getting-started.mdx
├── globals.css
├── layout.tsx
├── obfuscated-styles.css
└── token.css
scripts/
└── obfuscate-tailwind.js
```

## Build flow summary

1. You write readable Tailwind utility strings in your components.
2. The script finds every matching `className="..."` or `class="..."` string.
3. It replaces each full class string with a generated selector.
4. It writes the generated selector rules to `app/obfuscated-styles.css`.
5. Tailwind resolves the `@apply` output, including token-based classes, as long as `token.css` is imported and referenced correctly.
